# Contributing Notes

**Status:** Draft  
**Purpose:** Short contributor guide for the design repository

---

## What Counts As Useful Work

- Research findings with a clear design implication.
- RFC feedback that improves clarity or correctness.
- New RFC proposals that fit the vision and non-goals.
- Documentation updates that keep the repository coherent.

## Before You Start

- Read RFC-0001 to understand the language vision and non-goals.
- Check the existing RFCs so you do not duplicate work.
- Review the implementation plan so your contribution lands in the right place.

## Good Contribution Shape

- State the problem clearly.
- Show the design idea with a sketch or example.
- Explain how it aligns with the vision.
- List tradeoffs honestly.

## RFC Workflow

1. Open an issue to validate the idea.
2. Copy the RFC template.
3. Fill in every section.
4. Open a pull request.
5. Revise in response to feedback.
6. Accept or reject the RFC with a clear reason.

## Writing Style

- Use plain language where possible.
- Prefer concrete examples over abstract argument.
- Treat alternatives and drawbacks as required parts of the design.
